MEnvAgent

Scalable Polyglot Environment Construction for Verifiable Software Engineering

Automatically building executable environments is a critical challenge in software engineering research. Manual environment setup is time-consuming, error-prone, and often blocks the scalable generation of high-quality training data for software engineering tasks. MEnvAgent addresses this challenge by providing an automated multi-agent framework that constructs executable environments across 10 programming languages, enabling verifiable software engineering at scale.

Overview

MEnvAgent is designed to automate the challenging task of constructing executable environments for software repositories. Building on recent advances in large language models, our framework employs a multi-agent architecture where specialized agents collaborate to analyze repositories, select appropriate base images, install dependencies, configure tests, and verify environment correctness. This automation is critical for scaling up the generation of high-quality training data for software engineering tasks.

Key Capabilities

Multi-Language Support

Automated environment construction for 10 mainstream programming languages including Python (3.6-3.13), Java (8, 11, 17), TypeScript, JavaScript, Rust, Go, C++, Ruby, PHP, and C. Each language includes pre-built base images with common dependencies and build tools.

Multi-Agent Architecture

A Planning-Execution-Verification closed-loop system where specialized agents handle repository analysis, environment setup, test configuration, execution monitoring, and verification. Agents communicate through structured interfaces and can iterate until environment correctness is achieved.

Environment Reuse Mechanism

Instead of building environments from scratch for each repository, MEnvAgent retrieves similar historical environments from an environment pool and applies incremental patches. This approach reduces computational overhead by up to 43% while improving success rates.

Docker/Kubernetes Backend

Flexible container orchestration supporting both Docker for local development and Kubernetes for large-scale distributed execution. The abstraction layer allows seamless switching between backends based on workload requirements.

Fail-to-Pass Validation

Rigorous validation protocol ensuring that test cases fail on the buggy version of code and pass after applying the fix patch. This validates that environments correctly reproduce the original issues and fixes, essential for training verifiable software engineering agents.

High Performance

Achieves 8.6% improvement in Fail-to-Pass rates compared to baseline approaches while reducing time costs by 43%. The performance gains come from environment reuse, parallel execution, and intelligent error recovery mechanisms.

Architecture Overview

Multi-agent system structured into three iterative stages

Overview of MEnvAgent: (Top) Environment Reuse Mechanism retrieves and adapts historical environments. (Bottom) Planning-Execution-Verification loop with autonomous agents.

Planning Stage

  • Repository Analysis Agent: Explores repository structure and generates comprehensive summary
  • Environment Setup Agent: Determines suitable base image and generates installation scripts
  • Test Configuration Agent: Synthesizes compatible test configuration scripts

Execution Stage

  • Environment Execution Agent: Instantiates containers and executes build commands with real-time monitoring
  • Dynamic error resolution and automatic retry mechanisms

Verification Stage

  • Verification Agent: Executes tests and performs error attribution
  • Iterative refinement loop with diagnostic feedback to planning stage

Environment Reuse Mechanism

Instead of building environments from scratch, MEnvAgent:

  1. Retrieves similar historical environments from the Environment Pool
  2. Uses EnvPatchAgent to generate incremental patches
  3. Applies patches to adapt the retrieved environment to target repository
  4. Significantly reduces computational overhead and improves success rates
